产品定义
角色工作台 Character Workbench
以「角色」为第一公民的本地 AI 创作管理系统。每个角色是一个独立容器，绑定其全部多模态身份资产：视觉 LoRA、声音指纹、外貌参数、生成历史、造型版本树。用户在角色语境内完成创作，工作台自动保持跨场景一致性。
核心价值主张（一句话）
"你的角色，永远是同一个人。"
解决现有工具链的根本缺陷：图像生成工具只管生成，不管「是谁生成的」。

流程 A：创建新角色（首次使用）
1
命名角色 & 上传参考图
输入角色名称，上传 10–30 张参考图（支持拖拽批量导入）。
系统自动检测：图片分辨率、角度多样性、人脸清晰度，实时给出「数据集质量评分」和改进建议（如「缺少侧脸角度，建议补充」）。
2
填写角色 DNA（半自动）
系统自动分析参考图提取：发色、瞳色、肤色标签，用户确认或修正。手动补充：风格标签（二次元/写实/3D）、角色性格备注。
DNA 参数将作为所有生成任务的「基础 Prompt 锚点」，确保不同场景下生成词根一致。
3
一键 LoRA 训练
选择基础模型（FLUX-dev / SDXL），设置训练强度（轻量/标准/精细），点击「开始训练」。
后台调用 AI-Toolkit（kohya 引擎），显示实时进度条、预估剩余时间（RTX 3070 标准模式约 25–40 分钟）。训练中每 20% 进度生成一张采样预览图，让用户实时看到角色「逐渐成型」。
4
训练完成：角色卡片激活
LoRA 训练完成，系统自动生成 4 张「角色验证图」（正面、3/4 侧、背面、特写），供用户确认一致性。
用户对每张图「满意 / 不满意」评分。若不满意，系统推荐「增加训练步数」或「补充特定角度参考图」，支持一键重训。
5
（可选）绑定声音指纹
上传 5–30 秒角色声音参考音频（可以是自己配音或 TTS 生成），系统用 F5-TTS 提取声纹嵌入。
输入一段测试文字，立即试听克隆后的角色声音。确认后绑定到角色档案，之后所有 TTS 生成默认使用此声音。
流程 B：日常生产（内容创作）
1
选择角色 + 造型
从角色库选择目标角色，从造型版本树中选择造型（默认基础造型）。系统自动加载对应 LoRA + DNA prompt。
2
描述场景，不描述角色
用户只需输入场景描述，如「在咖啡馆看书，阳光从窗户射入」，无需重复描述角色外貌。
系统自动将「场景 Prompt」+ 「角色 DNA Prompt」+ 「LoRA 触发词」合并，构建完整生成指令。用户可展开查看完整 Prompt，但正常使用不需要关心。
3
生成 & 入库
生成结果自动归档到当前角色的生成历史，附带完整参数快照。用户添加用途标签（封面图/表情包/周边等）。
4
批量模式
输入 N 个场景关键词，系统排队批量生成，GPU 空闲时自动执行，完成后通知。
适合「一次准备一周直播封面图」的场景，后台挂机跑，不占用前台操作。
流程 C：创建造型分支
1
基于现有造型新建分支
在版本树上右键任意造型节点，选择「新建分支」，输入分支名如「万圣节版」。
2
用自然语言描述差异
输入「黑色女巫服、橙色腰带、戴南瓜帽」，系统将此作为造型附加 Prompt，叠加在基础角色 DNA 之上。
核心面部特征由基础 LoRA 保证不变，只有服装、配饰按描述变化。这是与从零重新生成的本质区别。
3
生成预览 & 确认
生成 4 张预览图，若满意则保存为正式造型版本；若不满意则修改造型描述重新生成，不影响基础造型。
多模态资产绑定架构
视觉层
FLUX LoRA + IP-Adapter
v1 核心
角色面部特征锁定。LoRA 保证深层一致性（90%+ 识别率），IP-Adapter 作为无训练快速参考补充。
声音层
F5-TTS / CosyVoice3
v1 核心
零样本声音克隆，3–30 秒参考音频即可提取声纹。生成 TTS 内容时自动使用绑定声音，无需每次指定。
动作层
AnimateDiff / WAN LoRA
v2 规划
角色动作风格绑定。训练角色专属的动作 LoRA，让视频帧中的角色动作风格也保持一致。

关键技术决策说明
为什么选 Tauri 而非 Electron
Tauri 使用系统 WebView（不打包 Chromium），安装包小 10 倍（~8MB vs ~80MB），内存占用低 50%，对于本地 AI 工具极为重要——省下的内存留给 GPU VRAM。前端复用现有 TypeScript/React 技能，迁移成本最低。
为什么用 SQLite 而非 Postgres
这是本地优先桌面应用，SQLite 是唯一正确选择：零配置、单文件、无需独立进程，随应用启动/关闭。Postgres 是 SaaS 服务端数据库，与「完全本地、无服务器」的产品承诺直接矛盾。
为什么对接 ComfyUI API 而非自建推理
ComfyUI 已经是生态标准，拥有最广泛的模型支持和自定义节点库。Mely 通过 ComfyUI 的 API 模式（无头启动，程序化调用）调用其推理能力，而不是让用户直接操作 ComfyUI 界面。这样既利用了成熟推理引擎，又提供了 Mely 自己的角色工作台体验。
最小化启动路径（Phase B → MVP）
①
替换数据层（1 周）
将现有 Postgres 替换为 SQLite，迁移 Character / Generation / Costume 三张核心表，清理服务端 PLpgSQL 代码。
②
接入 AI-Toolkit 训练后端（2 周）
Python FastAPI 包装 AI-Toolkit，实现「接收图片数据集 → 启动训练 → 流式返回进度」的 REST API。
③
接入 ComfyUI API 生成后端（1 周）
ComfyUI 无头模式启动，Mely 通过 websocket 调用，自动注入角色 LoRA + DNA Prompt，返回生成图片并入库。
④
接入 F5-TTS 声音后端（1 周）
本地部署 F5-TTS 服务，API 接收「文本 + 声纹嵌入向量」，返回音频文件并归档到角色声音历史。
⑤
打包 Tauri 桌面应用（1 周）
将 React 前端打包为 Tauri 桌面应用，Python 后端作为 sidecar 进程随应用启动，形成完整的独立桌面包。
MVP 交付标准（6 周）
用户能在不打开终端、不了解任何 AI 技术的情况下：上传照片 → 训练角色 LoRA → 生成 5 张一致性封面图 → 绑定声音 → 用角色声音朗读一段文字。整个流程在一个桌面窗口内完成，数据完全本地。
