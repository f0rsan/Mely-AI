# Mely AI — 项目全局上下文

> ⚠️ 每个 Codex 线程开始前必须读此文件。这是所有已锁定决策的唯一真相来源。

## 1. 产品定位

Mely AI 是一款**本地优先的桌面 AI 创作应用**（Windows-first，RTX 30/40 系列 GPU）。核心产品是「角色工作台 Character Workbench」——以「角色」为第一公民，绑定语言模型（LLM LoRA）、视觉 LoRA、声音指纹、人设文字、生成历史、造型版本树，让角色在**文字、视觉、声音**任意场景下保持**多模态一致性**。

**核心价值主张：「你的角色，永远是同一个人。」**

解决的根本问题：现有 AI 工具只管单次生成，不管「是谁在说话/出镜」。角色的性格、外貌、声音在不同工具间割裂，无法保持一致。

**创作循环（两条独立主链，可组合）**

| 链 | 入口 | 路径 | 核心体验 |
|----|------|------|---------|
| **链 A（文字优先）** | 描述角色 + 上传文字数据集 | 本地 LLM 微调 → 私有角色模型 | 与角色对话，验证人设一致性 |
| **链 B（视觉优先）** | 上传图片 或 文字生图 | LoRA 训练 → 图像生成 → 造型管理 | 生成跨场景一致视觉素材 |

两链通过「角色 DNA」共享人设参数：链 A 的 LLM 人设可自动填充链 B 的视觉 Prompt，互相增强。

## 2. 目标用户（优先级排序）

1. **Indie Vtuber 创作者**（核心用户）— 需要角色跨直播封面、素材、周边保持一致。高频使用、高付费意愿。
2. **IP / 漫画创作者** — 连载角色需要在数十章中保持视觉一致性。私密性要求高。
3. **独立游戏开发者** — 角色概念设计迭代快，需要版本分支管理和多视角素材批量生成。
4. **个人隐私创作者** — 不想数据上云，需要完全本地的私人角色创作环境。

## 3. 技术栈（已锁定 · 不可替换）

| 层 | 技术选型 | 选型理由 |
|----|---------|---------|
| 桌面框架 | Tauri 2.x + React/TypeScript | 安装包 ~8MB（vs Electron ~80MB），省内存留给 GPU |
| 后端服务 | Python FastAPI | sidecar 进程，随 Tauri 应用启动/关闭 |
| 数据层 | SQLite | 零配置、单文件、无需独立进程，本地优先的唯一正确选择 |
| LoRA 加密 | AES-256 | 绑定设备指纹，防止模型文件泄露 |
| 视觉训练 | AI-Toolkit（Kohya 兼容） | 支持 FLUX/SDXL LoRA 训练 |
| 图像生成 | ComfyUI API（无头模式） | 生态标准，最广泛的模型支持和自定义节点库 |
| 一致性补充 | IP-Adapter | 无训练快速参考 |
| 自动打标 | WD14 Tagger | 辅助角色 DNA 自动提取 |
| 声音克隆 | F5-TTS / CosyVoice3 / IndexTTS-2 | 零样本克隆，3–30 秒参考音频即可 |
| 默认模型 | FLUX-schnell 或 FLUX-dev | M1 第一周 PoC 后锁定 |

| 本地 LLM 推理 | Ollama | 管理基础模型下载与推理服务，对用户完全透明 |
| LLM 微调 | Unsloth + QLoRA | Apache 2.0，VRAM 效率 2x，支持 Qwen2.5/Llama3 |
| 基础 LLM | Qwen2.5-7B Q4_K_M | 中文优先，4-bit 量化，RTX 3070 推理 ~5GB VRAM |

### v1.5 追加（v1 验证后才引入）
| 层 | 技术选型 | 说明 |
|----|---------|------|
| 声音精训 | Unsloth TTS 微调 | 仅用 Apache 2.0 训练库，不集成 Studio UI (AGPL-3.0) |

### 关键分工边界
- **Ollama** → 本地 LLM 推理服务（加载基础模型 + 微调后的自定义模型）
- **Unsloth** → LLM QLoRA 微调（合并导出 GGUF → 交给 Ollama 推理）
- **AI-Toolkit** → 图像扩散模型 LoRA 训练（FLUX/SDXL）
- 三者分工明确，互不干扰

## 6. 编码规范

- **前端**: TypeScript strict mode, React functional components + hooks, Tailwind CSS
- **后端**: Python 3.11+, type hints everywhere, Pydantic models for all API schemas
- **代码注释**: 英文
- **Commit message**: `[M{N}] {简短英文描述}`, e.g. `[M0] Add SQLite schema migration`
- **UI 文本**: 中文（后续做 i18n）
- **测试**: TDD，每个功能先写测试
- **错误处理**: 所有用户可见的错误信息用中文自然语言，禁止暴露 Python traceback

## 7. 已识别的高风险项

| 风险 | 缓解策略 | 涉及模块 |
|------|---------|---------|
| Unsloth QLoRA 在 RTX 3070 8GB 上的 VRAM 上限 | 三档预设严格控制在 7GB 以内，训练前检测可用 VRAM | M1 |
| Ollama 自定义 GGUF 模型注册兼容性 | Unsloth 合并导出后验证 Ollama 加载，记录兼容版本 | M1 |
| LLM 训练数据质量（用户不懂格式） | 自动格式转换 + 质量检测 + 中文改进建议，降低门槛 | M1 |
| ComfyUI 无头调用稳定性 | M3 前必须验证，准备 diffusers fallback | M3 |
| RTX 3070 VRAM 多任务冲突 | 全局 GPU 互斥锁，任意时刻只跑一个 AI 任务 | M1/M2/M3 |
| 安装体积（LLM 模型 ~4.5GB + 视觉模型 ~5GB） | 按需分级下载，用户可选只装 LLM 或只装视觉 | M0/M6 |
| 训练失败路径 | 所有技术错误翻译成用户语言（LLM + 视觉双覆盖） | M1/M2 |
