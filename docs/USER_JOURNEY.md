# Mely AI — Vtuber 用户旅程深度分析

> 核心用户画像「小A」：25岁，有全职工作，业余做 Vtuber。已有 1200 粉丝，目标月收入覆盖创作成本。有基础审美，无编程/3D建模能力。GPU: RTX 3070。

## Phase 1: 创建角色（从零到形象）

### 当前痛点
- **极高**: 立绘和 Rigging 两次独立委托，两个风格/价格/交付周期，极难保持视觉统一性
- **极高**: 委托周期 2–8 周，期间无法开播，初始投入 $500–$3000
- **中等**: 用 SD 生成立绘后，角色在不同光影/角度下一致性差
- **中等**: 没有统一管理「角色设定文件」的地方

### Mely 机会点
- 一键从参考图训练「角色 LoRA」，锁定外貌一致性
- 角色卡片：统一存储设定（外貌参数 + LoRA + 触发词 + 参考图谱）
- 从角色 LoRA 直接生成多角度参考图，辅助委托画师时精确传达需求

## Phase 2: 内容生产（日常素材制作）

### 当前痛点
- **极高**: 「角色一致性」是最大痛点——每次生成封面，发色/瞳色/服装都容易漂移
- **极高**: 生产效率极低——一张合格封面需反复生成 20–50 张，每次直播准备 2–4 小时
- **中等**: Prompt 管理混乱，靠记忆或 txt 文件存储
- **中等**: 表情包要求「和立绘一致」但无法调用立绘精确特征

### Mely 机会点
- 「角色场景模板」：绑定 LoRA + 场景 Prompt，一键生成
- 批量生成模式：10 个主题关键词 → 自动批量输出
- 素材历史库：所有生成图按角色 + 用途归档

## Phase 3: 角色进化（换装/新表情）

### 当前痛点
- **极高**: 每次换装 = 重新走委托流程，$300–$800，3–6 周
- **极高**: 新服装必须与原角色面部高度一致，两次委托几乎不可能统一
- **中等**: 预热宣传需要新旧造型对比，但旧参数已丢失

### Mely 机会点
- 「造型分支」：基于同一角色 LoRA 新建服装变体，3 分钟出预览
- 角色版本历史：存储每版造型参数，随时复原
- 换装预告包：一键生成「旧装 vs 新装」对比图组

## Phase 4: IP 变现（周边/授权）

### 当前痛点
- **极高**: 没有标准化的「角色设定书」输出能力
- **极高**: AI 生成图商用版权不明确
- **中等**: 周边物料需多视角图，AI 生成角度一致性差

### Mely 机会点
- 一键导出「角色设定书」PDF
- 周边素材包生成：批量标准视角图
- 商用 LoRA 证明：元数据存档作为版权凭证

## Phase 5: 资产保护（版权/防盗）

### 当前痛点
- **极高**: 私人 LoRA 无保护机制，文件流出则任何人可生成相似内容
- **极高**: 缺乏创作时间戳，无法证明原创优先权
- **中等**: 角色设定散落多个工具

### Mely 机会点
- 本地 LoRA 加密存储：绑定设备，防泄露
- 创作时间戳：哈希值 + 时间 → 可导出创作证明
- 角色 DNA 档案：不可篡改的本地记录
