# Mely AI — Impeccable Design Context

## Design Context

### Users
**Indie creators in CJK market** — Vtubers, IP/manga artists, indie game devs.  
Primary context: working alone at a GPU workstation late at night, iterating on a single beloved character across many creative formats (stream thumbnails, merch, game sprites, voice lines).  
Core job-to-be-done: *lock a character's identity once, then generate forever without re-describing her.*  
Emotional state during use: focused, slightly impatient, proud of their IP. They care deeply about their characters feeling "right."

### Brand Personality
**Three words: Precise · Alive · Yours**  
- *Precise* — every parameter is explicit, every generation traceable. This is a craft tool, not a toy.  
- *Alive* — the interface should feel like the character is present: animations, glow, pulse.  
- *Yours* — each character gets her own color world. The UI bends to the character, not the other way around.

**Tagline:** 你的角色，永远是同一个人。  
**Voice:** Confident and technical, but written in warm Chinese. Never cold or enterprise-y.

### Aesthetic Direction
**Reference:** The `prototypes/mely-unified.jsx` artifact — cold dark base, per-character accent colors, fine micro-animations, SVG icon set, gradient cover art placeholders.  
**Anti-references:** Generic SaaS dashboards, light-mode admin panels, anything that looks like a Tailwind template.

**Core palette (from prototype):**
- Background: `#0A0A0F` with subtle radial orbs (`hsla(280,50%,25%,0.06)` purple, `hsla(200,45%,20%,0.04)` teal)
- Surface: `rgba(255,255,255,0.02–0.04)` layered glass
- Border: `rgba(255,255,255,0.04–0.08)` default, accent-tinted on hover
- Global accent: `#FF6B9D` (pink) + `#7B68EE` (indigo) gradient for brand elements
- Per-character accent: deterministic hue from character ID (front-end only, no backend field)
- Text primary: `rgba(255,255,255,0.92)`
- Text secondary: `rgba(255,255,255,0.35–0.45)`
- Monospace: JetBrains Mono (prompt preview, technical values)
- Success: `#5DCAA5` / Error: `#E94560` / Warning: `#FF8C42`

**Typography:**
- Display: DM Sans 700, letter-spacing -0.03em
- Body: DM Sans 400/500 + Noto Sans SC (CJK fallback)
- Code/prompt: JetBrains Mono 400/500
- Load via Google Fonts in `index.html`

**Motion:**
- `cardIn`: `opacity 0 → 1, translateY 20px → 0, scale 0.96 → 1`, 600ms cubic-bezier(0.16,1,0.3,1)
- `pageIn`: `opacity 0 → 1, translateX 12px → 0`, 400ms cubic-bezier(0.16,1,0.3,1)
- `fadeIn`: `opacity 0 → 1, translateY 6px → 0`, 300ms ease-out
- `shimmer`: gradient background-position sweep, 6s linear infinite (hero headline)
- `pulseRing`: border-radius ring scale + fade, 2s ease-in-out infinite (training status)
- `spin`: 1s linear infinite (loading spinners)
- Hover transitions: 200–400ms cubic-bezier(0.19,1,0.22,1)

### Layout Architecture
**Shell:** Full-viewport dark canvas (`height: 100vh`, `overflow: hidden`), no padding.  
**Nav:** 56px icon-only sidebar, left edge. Logo at top, page icons below with tooltip-on-hover.  
**Content:** `flex: 1`, `overflow: hidden`, receives page transitions.

**Page hierarchy:**
1. **Home** — character grid + hero text + GPU status badge + search/filter
2. **Detail** — 230px left panel (char card + vertical tabs) + right content area
3. **Generate** — 360px left input panel + right results grid
4. **Train** — single scrollable page (progress + loss curve + sampling previews)

**Detail tabs (5 user-facing, dev features integrated within):**
| Tab | Icon | Integrated dev content |
|-----|------|------------------------|
| 角色 DNA | DNA | DNA form + dataset evaluation entry |
| 视觉工作台 | Image | Visual training + dataset import + text-to-character |
| LLM 工作台 | Bolt | LLM training + chat |
| 生成工作台 | Generate | Generation + batch queue |
| 声音 & 导出 | Voice | Voice bind + TTS + costume tree + export |

**Character accent color:** `hue = (charId.charCodeAt(0) * 137 + charId.charCodeAt(n) * 31) % 360`, then `hsl(hue, 70%, 65%)`. Injected as `--char-accent` CSS variable on the detail container; Tailwind and custom CSS both consume it.

### Confirmed Architecture Decisions
- **D1 Style system:** CSS Variables (`--char-accent`, `--surface`, etc.) + Tailwind utilities. No full inline-style migration.
- **D2 Tab merge:** Dataset evaluation + text-to-character live inside **角色 DNA** tab as a data-source sub-section (not Visual tab).
- **D3 styles.css:** Full token rewrite — all hard-coded color literals replaced with CSS custom properties. Single source of truth.

### Design Principles
1. **Character-first color** — every detail page, progress bar, glow, and border draws from that character's accent. The tool disappears; the character takes over.
2. **Show the machine working** — GPU temp, VRAM usage, loss curve, step count. Creators trust tools that are transparent about what they're doing.
3. **Describe the scene, not the character** — the generation input only asks for context (场景). DNA + LoRA handle the character automatically. The UI must reinforce this mental model.
4. **One task at a time** — global GPU mutex means only one AI job runs at once. The UI must make this constraint feel like safety, not limitation.
5. **Animate with purpose** — every animation communicates state (training pulse = alive, cardIn stagger = collection, pageIn slide = navigation depth). No decorative motion.
