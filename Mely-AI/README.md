# Mely AI Workspace

使命：做每个创作者的私人模型资产平台：**能创作、能进化、能交易、可治理**。

## 执行原则
- 默认直接开发，只有关键决策点/风险点再向你问询
- 所有中间节点文档统一沉淀到本目录
- 多角色并行协作、交叉评审、相互监督

## 角色分工
- 产品经理（PM）
- UI/UX
- 前端开发
- 后端开发
- 数据库工程师
- 算法工程师
- 模型工程师
- 测试工程师
- 项目经理（Delivery）

## 目录说明
- `00-Governance`：目标、边界、决策记录
- `01-Product`：PRD、功能拆解、验收标准
- `02-UX`：信息架构、交互稿、设计规范
- `03-Frontend`：前端架构、任务、实现说明
- `04-Backend`：后端架构、API、服务拆解
- `05-Database`：ERD、DDL、迁移脚本
- `06-ML`：算法策略、评估指标
- `07-Model`：模型接入、网关、推理策略
- `08-QA`：测试计划、用例、缺陷追踪
- `09-PM`：排期、里程碑、风险管理
- `10-Deliverables`：阶段交付物
- `logs`：开发日志、日更记录
