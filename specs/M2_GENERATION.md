# M2 生成工作台 — W8–10（含 1 周 buffer）

> 目标: ComfyUI 接入, Prompt 自动组装, 批量生成, 图库归档

## 前置条件
- M0 + M1 完成
- ComfyUI 已安装（按需下载框架提供）
- 至少一个角色已完成 LoRA 训练

## 任务拆解

### W8: ComfyUI 接入

#### T2.1 — ComfyUI 无头启动管理 (2d)
- Python 服务管理 ComfyUI 进程生命周期
- 无头模式启动（无 UI）
- 健康检查（HTTP ping）
- 崩溃自动重启（最多 3 次，之后通知用户）
- GPU 互斥锁: 训练进行时不启动 ComfyUI
- 验收: ComfyUI 自动启动 → 崩溃 → 自动重启 → 恢复正常

#### T2.2 — ComfyUI API 通信层 (2d)
- WebSocket 连接 ComfyUI API
- 工作流模板: 文生图、图生图
- 动态参数注入: LoRA 路径 + 权重 + Prompt
- 生成进度实时推送
- 生成完成 → 取回图片文件
- 验收: 通过 API 提交一个文生图任务 → 拿到结果图片

#### T2.3 — Prompt 自动组装引擎 (1d)
- 输入: 用户场景描述（如「在咖啡馆看书」）
- 自动合并: 场景 Prompt + 角色 DNA Prompt + LoRA 触发词 + 造型附加 Prompt
- 合并优先级和冲突处理规则
- 用户可展开查看/编辑完整 Prompt（高级模式）
- 验收: 输入场景描述 → 看到合并后的完整 Prompt → 可选编辑

### W9: 生成体验 + 批量模式

#### T2.4 — 单张生成 UI (2d)
- 选择角色 + 造型（顶部选择器）
- 场景描述输入框
- 生成按钮 → 实时进度 → 结果预览
- 参数面板（可折叠）: seed, steps, sampler, CFG, 图片尺寸
- 「再来一张」按钮（换 seed 重新生成）
- 验收: 完整的 选角色 → 输入场景 → 生成 → 看结果 流程

#### T2.5 — 批量生成队列 (2d)
- 输入 N 个场景关键词（textarea，每行一个）
- 提交 → 任务队列排队
- 队列面板: 进行中/等待中/已完成
- GPU 空闲时自动执行下一个
- 全部完成 → 系统通知（Windows toast）
- 验收: 输入 5 个场景 → 提交 → 后台依次生成 → 全部完成通知

#### T2.6 — 生成结果归档 (1d)
- 生成结果自动存入 generations 表
- 完整参数快照（seed, steps, sampler, LoRA weight 等）
- 添加用途标签（封面图/表情包/周边/预告图）
- 文件自动归档到 `~/.mely/characters/{id}/generations/`
- 验收: 生成 → 自动入库 → 能在历史中找到并查看参数

### W10: Buffer + 稳定性

#### T2.7 — diffusers Fallback 实现 (1–2d)
- 如果 ComfyUI 反复崩溃 → 自动切换到 diffusers 直接推理
- 基本文生图能力（不要求完整 ComfyUI 节点支持）
- 用户通知: 「图像引擎已切换到备用模式」
- 验收: 模拟 ComfyUI 崩溃 3 次 → 自动 fallback → 仍能生成图片

#### T2.8 — 图库浏览 UI (1d)
- 角色详情页 → 生成历史 tab
- 缩略图网格
- 筛选: 按造型 / 按标签 / 按日期
- 点击 → 大图预览 + 生成参数详情
- 「用此参数重新生成」按钮
- 验收: 能浏览角色的所有历史生成，筛选有效

#### T2.9 — 端到端测试 (1d)
- 完整流程: 选角色 → 输入场景 → 生成 → 归档 → 浏览历史
- 批量流程: 输入多场景 → 排队 → 全部完成
- 异常: ComfyUI 崩溃 → 恢复 / fallback
- 验收: 主路径 + 异常路径全部跑通
