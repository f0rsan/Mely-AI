# M3 声音绑定 — W11–12

> 目标: F5-TTS 集成, 声纹绑定, TTS 生成归档

## 前置条件
- M0 + M1 完成
- F5-TTS / CosyVoice3 模型可用（按需下载）

## 任务拆解

### W11: TTS 服务 + 声纹绑定

#### T3.1 — F5-TTS 本地服务 (2d)
- F5-TTS 模型按需下载
- Python 封装 TTS 推理服务
- API: `POST /api/voice/synthesize` → 接收文本 + 声纹嵌入 → 返回音频文件
- GPU 互斥（与训练/生成不同时跑）
- 支持 CosyVoice3 作为备选引擎
- 验收: 调用 API → 传入文本 + 参考音频 → 返回克隆后的音频

#### T3.2 — 声纹提取 & 绑定 (1d)
- 上传 5–30 秒参考音频
- 提取声纹嵌入向量（F5-TTS 内置功能）
- 保存到 voice_assets 表 + 本地文件
- 验收: 上传参考音频 → 声纹提取成功 → 存储到角色档案

#### T3.3 — 声音绑定 UI (2d)
- 上传参考音频（拖拽或文件选择）
- 音频波形预览
- 测试文字输入框 → 「试听」按钮 → 播放克隆后的声音
- 满意 → 「确认绑定」→ 存储到角色
- 不满意 → 换参考音频重试
- 验收: 完整的 上传 → 试听 → 确认绑定 流程

### W12: TTS 生成 + 归档

#### T3.4 — 角色 TTS 生成 (1d)
- 角色详情页 → 声音 tab
- 输入文字 → 选择语言/语速
- 生成音频 → 播放预览
- 导出音频文件（wav/mp3）
- 验收: 输入文字 → 用角色声音输出 → 可播放可导出

#### T3.5 — 音频归档 (1d)
- TTS 生成结果存入 generations 表（type=audio）
- 参数快照: 输入文本、引擎、采样率
- 文件归档到角色目录
- 历史音频列表 + 回放
- 验收: 生成音频 → 自动入库 → 历史中能找到并回放

#### T3.6 — 集成测试 (1d)
- 完整流程: 绑定声音 → 生成 TTS → 归档 → 回放历史
- 边界: 参考音频太短（<3秒警告）、无 GPU（提示需要 GPU）
- 验收: 主路径 + 边界情况跑通
