# M4 造型&导出 — W13–15

> 目标: 造型版本树, 设定书 PDF 导出, 创作时间戳

## 前置条件
- M0 + M1 + M2 完成
- 至少一个角色有完成的 LoRA + 生成历史

## 任务拆解

### W13: 造型版本系统

#### T4.1 — 造型 CRUD API (1d)
- `POST /api/characters/{id}/costumes` — 创建造型分支
- `GET /api/characters/{id}/costumes` — 列表（含树结构）
- `PUT /api/costumes/{id}` — 更新造型描述/名称
- `DELETE /api/costumes/{id}` — 删除造型（不影响父/兄弟节点）
- 验收: API 测试全通过，树结构正确

#### T4.2 — 造型分支创建流程 (2d)
- 版本树上右键任意造型节点 → 「新建分支」
- 输入分支名（如「万圣节版」）
- 用自然语言描述差异（如「黑色女巫服、橙色腰带、戴南瓜帽」）
- 系统将描述作为 costume_prompt，叠加在基础角色 DNA 之上
- 自动生成 4 张预览图
- 满意 → 保存为正式造型版本
- 不满意 → 修改描述重新生成
- 验收: 完整的 新建分支 → 描述 → 预览 → 确认 流程

#### T4.3 — 版本树可视化 UI (2d)
- 树状展示所有造型及父子关系
- 每个节点显示: 造型名 + 缩略预览图
- 节点操作: 选为当前造型 / 新建分支 / 删除
- 基础造型标记为根节点（不可删除）
- 验收: 能清晰看到造型树结构，交互操作正确

### W14: 设定书导出

#### T4.4 — 设定书数据聚合 (1d)
- 聚合角色全部资产:
  - 角色名称 + 创建时间
  - DNA 参数表
  - 多角度参考图（正面/3-4 侧/背面/特写）
  - 造型列表（每个造型含预览图 + 描述）
  - LoRA 基本信息（触发词、推荐权重，不含模型文件）
- JSON 中间格式，前后端共用
- 验收: API 返回完整的设定书数据 JSON

#### T4.5 — PDF 设定书生成 (2d)
- 使用 Python PDF 库（reportlab 或 weasyprint）
- 排版: 封面 → DNA 参数页 → 多角度参考图页 → 造型列表页
- 中文字体支持（Noto Sans CJK）
- 输出到 `~/.mely/characters/{id}/exports/`
- 验收: 导出的 PDF 排版美观，图片清晰，中文正常显示

#### T4.6 — 导出 UI (1d)
- 角色详情页 → 导出 tab
- 「导出设定书 PDF」按钮
- 预览 → 确认 → 生成 → 打开文件位置
- 验收: 一键导出 PDF 到本地

### W15: 创作时间戳 + 集成

#### T4.7 — 创作时间戳系统 (2d)
- 每次训练/生成自动记录:
  - 操作类型（训练/图像生成/TTS 生成）
  - 时间戳（ISO 8601）
  - 数据哈希（训练数据集 SHA-256 / 生成参数 SHA-256）
  - 输出文件哈希
- 哈希链: 每条记录包含前一条的哈希（简易区块链思路）
- `POST /api/characters/{id}/proof` → 导出创作证明 JSON 文件
- 验收: 训练 + 生成后有完整时间戳链，导出的 JSON 可验证

#### T4.8 — LoRA 加密存储 (1d)
- AES-256 加密 LoRA safetensors 文件
- 密钥绑定设备指纹（CPU ID + 磁盘序列号）
- 加密/解密透明化（加载时自动解密到内存）
- 验收: LoRA 文件在磁盘上加密，复制到其他设备无法使用

#### T4.9 — M4 集成测试 (2d)
- 造型流程: 创建分支 → 预览 → 确认 → 切换造型生成
- 导出流程: 聚合 → 预览 → PDF 导出
- 时间戳: 训练 → 生成 → 导出证明 → 验证哈希链
- 加密: 加密存储 → 正常加载 → 文件复制到外部无法使用
- 验收: 全部主路径跑通
