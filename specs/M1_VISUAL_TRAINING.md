# M1 视觉训练 — W4–7（含 1 周 buffer）

> 目标: 数据集管理, AI-Toolkit LoRA 训练, 进度可视化, 文字创角入口

## 前置条件
- M0 全部完成
- AI-Toolkit 可用（pip install）
- RTX 3070 8GB 开发机

## 任务拆解

### W4: PoC + 数据集管理

#### T1.1 — FLUX 模型 PoC (2d) ⚠️ 关键决策点
- RTX 3070 8GB 上测试:
  - FLUX-dev Q4 量化: 训练能否跑通？VRAM 是否溢出？
  - FLUX-schnell: 训练质量对比
- 输出: 选型报告，锁定默认模型
- 验收: 有明确结论文档，选定模型在 8GB VRAM 内可训练

#### T1.2 — 数据集导入 UI (2d)
- 拖拽批量导入图片（前端 dropzone）
- 图片预览网格
- 自动检测: 分辨率、人脸清晰度（用 OpenCV face detection）
- 数据集质量评分（简单规则: 图片数量 + 分辨率均值 + 人脸检出率）
- 改进建议文字（如「缺少侧脸角度，建议补充」）
- 验收: 拖入 15 张图 → 显示质量评分 → 有具体改进建议

#### T1.3 — WD14 自动打标集成 (1d)
- 集成 WD14 Tagger（按需下载模型）
- 对导入图片自动打标
- 标签作为 DNA 填写时的「建议值」
- 验收: 导入参考图 → 自动生成标签建议 → 显示在 DNA 表单中

### W5: 训练管线

#### T1.4 — 角色 DNA 配置 UI (1d)
- 5 个下拉: 发色/瞳色/肤色/风格/体型
- WD14 建议值自动填充（用户可修正）
- 自动组合生成 `auto_prompt`
- 保存到 character_dna 表
- 验收: 填写 DNA → 预览自动生成的 Prompt → 保存

#### T1.5 — AI-Toolkit 训练后端 (3d)
- Python FastAPI 封装 AI-Toolkit
- REST API: `POST /api/training/start` → 接收角色ID + 训练模式
- 训练配置生成（根据轻量/标准/精细模式）
- 实时进度推送（WebSocket）
- 每 20% 进度调用 ComfyUI 生成一张采样预览图
- GPU 占用互斥锁（训练期间禁止其他 GPU 任务）
- 训练完成后自动更新 visual_assets 表
- 错误处理: VRAM 溢出 → 「显存不足，请尝试轻量模式」
- 验收: 上传 15 张图 → 选标准模式 → 训练完成 → LoRA 文件生成

#### T1.6 — 训练进度 UI (1d)
- 进度条 + 百分比
- 预估剩余时间
- 采样预览图实时展示（每 20% 一张）
- 训练完成 → 4 张验证图（正面/3-4 侧/背面/特写）
- 「满意/不满意」评分按钮
- 验收: 训练全程能看到进度 + 采样图

### W6: 文字创角 + 验证流程

#### T1.7 — 文字创角入口 (2d)
- 用户输入角色文字描述（如「银色长发、红色眼睛的女孩」）
- 调用 ComfyUI 文生图 → 生成 4–8 张候选图
- 用户选择满意的图 → 加入训练数据集
- 与手动上传图片路径合流（选完进入正常训练流程）
- 验收: 输入描述 → 看到候选图 → 选择 → 进入训练

#### T1.8 — 训练验证 & 重训流程 (1d)
- 训练完成 → 自动生成 4 张标准验证图
- 用户对每张图评分（满意/不满意）
- 不满意 → 推荐「增加训练步数」或「补充参考图」
- 一键重训按钮（保留数据集，调整参数）
- 验收: 完整的训练 → 验证 → 重训循环

### W7: Buffer + 集成测试

#### T1.9 — 端到端集成测试 (2–3d)
- 完整流程: 创建角色 → 上传图片 → DNA 配置 → 训练 → 验证
- 文字创角流程: 输入描述 → 选图 → 训练 → 验证
- 边界情况: 训练中断、VRAM 不足、图片数不够
- 性能: RTX 3070 上标准模式训练时间 < 40 min
- 验收: 两条主要路径全部跑通，错误路径有用户友好提示
