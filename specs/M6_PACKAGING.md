# M6 打包发布 — W21–23

> 目标: 安装引导, 新手 Onboarding（含 LLM 和视觉双链路）, 用户测试, Windows 打包

## 前置条件
- M0–M5 全部完成
- 完整功能可在开发模式下跑通

## 任务拆解

### W21: 安装体验

#### T6.1 — Windows 安装包构建 (2d)
- Tauri 打包 Windows MSI/NSIS 安装包
- Python 后端打包（PyInstaller，sidecar 模式）
- App 本体 < 100MB
- 安装后桌面快捷方式 + 开始菜单
- 卸载清理（可选保留用户数据 `~/.mely/`）
- 验收: 在干净 Windows 机器上安装 → 启动成功

#### T6.2 — 首次启动引导 (2d)
- 检测 GPU（NVIDIA RTX 30/40 系列）+ 检测 Ollama 是否已安装
- 不满足最低要求 → 友好提示（不是 Python traceback）
- 按需下载引导（用户可选模块）:
  - 步骤 1: 基础 LLM 模型 Qwen2.5-7B Q4（~4.5GB）— 必选（支持文字创角）
  - 步骤 2: 基础视觉模型 FLUX-schnell（~5GB）— 必选（支持图像生成）
  - 步骤 3: 训练引擎 AI-Toolkit（~1GB）— 可选（支持 LoRA 训练）
  - 步骤 4: TTS 模型 F5-TTS（~2GB）— 可选（支持声音绑定）
- 下载进度 + 断点续传
- 全部就绪 → 进入主界面
- 验收: 全新安装 → 引导下载 → 进入可用状态

#### T6.3 — GPU 检测 & 资源管理 (1d)
- 检测 GPU 型号、VRAM 大小
- 根据 VRAM 自动推荐训练模式:
  - 8GB → LLM 轻量/标准微调，视觉轻量/标准模式
  - 12GB+ → 全部模式
  - < 8GB → 警告，仅支持推理（LLM 对话 + 图像生成，不支持训练）
- 运行时 VRAM 监控（训练/生成时显示用量）
- GPU 互斥锁状态可视化（正在跑哪个 AI 任务）
- 验收: 不同 GPU 环境下推荐正确，VRAM 监控实时更新

### W22: 新手 Onboarding

#### T6.4 — 新手引导流程 (2d)
- 首次使用交互式引导（非视频教程），支持两条路径：

  **路径 A（文字优先）**
  - Step 1: 创建角色（输入名称 + 基础人设描述）
  - Step 2: 上传对话样本或人设文档
  - Step 3: 开始 LLM 轻量微调（引导等待训练完成）
  - Step 4: 和角色说第一句话

  **路径 B（视觉优先）**
  - Step 1: 创建角色（输入名称）
  - Step 2: 上传参考图（引导拖拽）
  - Step 3: 开始视觉轻量训练
  - Step 4: 生成第一张角色图

- 每步有「跳过」和「稍后再来」
- 完成后 → 进入角色库主页
- 验收: 完全不了解 AI 的用户能跟着引导完成第一次创作

#### T6.5 — 错误信息国际化 (1d)
- 审查所有后端错误信息（LLM 训练 + 视觉训练 + 图像生成 + 声音）
- 常见错误映射表:
  - CUDA OOM → 「显存不足，请尝试轻量模式或关闭其他使用 GPU 的程序」
  - LLM training failed → 「语言模型训练失败，可能原因：数据集条目过少（建议 50 条以上）」
  - ComfyUI crash → 「图像引擎遇到问题，正在自动重启...」
  - Download failed → 「下载中断，将在网络恢复后自动继续」
  - Ollama not found → 「未检测到语言引擎，请点击此处安装 Ollama」
- 验收: 所有用户可见的错误信息都是中文自然语言

#### T6.6 — 设置页面 (1d)
- 模型存储路径配置
- GPU 选择（多卡环境）
- 默认训练模式（LLM 和视觉分开配置）
- ComfyUI 路径（高级用户可自定义）
- Ollama 路径（高级用户可自定义）
- 语言切换（预留 i18n 接口，v1 只有中文）
- 验收: 设置保存持久化，重启后生效

### W23: 用户测试 + 修复

#### T6.7 — 用户测试 (3d)
- 招募 3–5 位目标用户（Vtuber 创作者 + IP 创作者）
- 测试场景:
  - 全新安装 → onboarding（路径 A）→ 创建角色 → LLM 微调 → 和角色对话
  - 全新安装 → onboarding（路径 B）→ 创建角色 → 视觉训练 → 生成封面
  - 造型分支 → 换装 → 批量生成 → 导出设定书
  - 声音绑定 → TTS
- 记录: 操作路径、卡点、困惑、完成时间
- 输出: Bug 列表 + 体验问题列表 + 优先级排序

#### T6.8 — Bug 修复 & 打磨 (2d)
- 修复用户测试发现的 P0/P1 bug
- UI 细节打磨（加载状态、空状态、边界 UI）
- 性能优化（首次加载、页面切换）
- 验收: P0 bug 全部修复，关键流程无卡顿
